# 技能評價清單功能設計文件

**日期**：2026-03-11
**功能**：在賽道資訊分析頁籤新增「技能評價清單」Popup

---

## 需求概述

在賽道資訊分析（Analysis Tab）中，用戶選擇賽道與跑法後，可點擊「技能評價」按鈕，以 Popup 形式顯示該賽道所有技能，依類型分組並按效率排序。

---

## 技能分類規則

共 6 個分類，判斷優先序由上至下：

| 類別鍵值 | 顯示名稱 | 判斷條件 | 說明 |
|---|---|---|---|
| `inherited` | 繼承技能 | `id >= 900000` | id 以 9xxxxx 開頭，memo 欄位含來源角色 |
| `unique` | 固有技能 | `iconId` 10011–10062（綠底） | 賽道/條件固有加成技能 |
| `accel` | 加速技能 | `iconId` 20041 或 20042（橙底無火） | 純加速效果技能 |
| `speed` | 速度技能 | `iconId` 20011 或 20012（橙底+火焰） | 純速度效果技能 |
| `combined` | 複合型技能 | `iconId` 20051/20052（鞋+箭頭）或 20021/20022（藍底）| 速度+加速或含恢復效果 |
| `other` | 其他 | 以上均不符合 | 未知 iconId，保留分類備用 |

### classifySkill 函數邏輯

```javascript
function classifySkill(skill) {
    const id = skill.id;
    const iconId = skill.iconId;

    if (id >= 900000)                                       return 'inherited';
    if (iconId >= 10011 && iconId <= 10062)                 return 'unique';
    if ([20041, 20042].includes(iconId))                    return 'accel';
    if ([20011, 20012].includes(iconId))                    return 'speed';
    if ([20051, 20052, 20021, 20022].includes(iconId))      return 'combined';
    return 'other';
}
```

### 分類規則依據（驗證資料）

- **繼承技能**：紅焔ギア/LP1211-M (id: 900041)、夏空ハレーション (id: 910321)、白い稲妻見せたるで！(id: 900211)
- **固有技能**：左回り◎ (iconId: 10011)、右回り○ (iconId: 10011)
- **加速技能**：一番乗り (iconId: 20041)、盤石の構え (iconId: 20042)
- **速度技能**：十全十美 (iconId: 20012)、尻尾上がり (iconId: 20011)
- **複合型技能**：ライトニングステップ (iconId: 20052)、連鎖反応 (iconId: 20022)

## 實作方案

**選擇方案 C：純前端分類 + 工具函數**

- 不修改 server.js，不修改現有 JSON 格式
- 新增 `classifySkill()` 工具函數於 `public/index.html`
- 分類結果快取於 Map，同賽道不重複計算

## UI 設計

### 觸發入口

- 位置：B 區（跑法選擇）下方，新增「📊 技能評價」按鈕
- 狀態：未選擇賽道+跑法時 `disabled`

### 技能列項目

每個技能列顯示：
- 技能圖示（`<img>` 48px）
- 技能名稱（套用效率顏色規則）
- 效率值 `X.XX [バ/Pt]`（套用效率顏色規則）
- 繼承技能額外顯示 `memo` 欄位（來源角色，灰色小字）

### 效率顏色規則

- 各分類**獨立計算**效率排名與顏色分級
- 沿用現有 `calculateEfficiencyTiers()` 函數
- 效率為 0 的技能排在各分類最後
- 空分類不顯示

### 互動

- 點擊背景遮罩或 ✕ 按鈕關閉 Popup
- Popup 開啟時顯示全部賽道技能（`availableSkills`，非 `selectedSkills`）
